    MCP Tools Reference¶

    Complete reference for all MCP tools available in MCP DevBench.

    Container Management Tools¶

    spawn¶

    Create a new Docker container.

    Input: 

    {
  "image": "python:3.11-slim",
  "persistent": false,
  "alias": "my-workspace",
  "ttl_s": 3600,
  "idempotency_key": "optional-key"
}

    Output: 

    {
  "container_id": "c_abc123...",
  "alias": "my-workspace",
  "status": "running"
}

    Parameters: - image (required) - Docker image reference - persistent (optional) - Whether container persists across restarts (default: false) - alias (optional) - User-friendly name for the container - ttl_s (optional) - Time-to-live in seconds - idempotency_key (optional) - Key to prevent duplicate creation

    attach¶

    Attach client session to a container.

    Input: 

    {
  "target": "my-workspace",
  "client_name": "claude",
  "session_id": "unique-session-id"
}

    Output: 

    {
  "container_id": "c_abc123...",
  "alias": "my-workspace",
  "roots": ["workspace:c_abc123..."]
}

    kill¶

    Stop and remove a container.

    Input: 

    {
  "container_id": "c_abc123...",
  "force": true
}

    Output: 

    {
  "status": "stopped"
}

    list¶

    List all containers.

    Input: 

    {
  "include_stopped": false
}

    Output: 

    {
  "containers": [
    {
      "container_id": "c_abc123...",
      "alias": "my-workspace",
      "image": "python:3.11-slim",
      "status": "running",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ]
}

    Execution Tools¶

    exec_start¶

    Start command execution in a container.

    Input: 

    {
  "container_id": "c_abc123...",
  "cmd": ["python", "script.py"],
  "cwd": "/workspace",
  "env": {"DEBUG": "true"},
  "timeout_s": 300
}

    Output: 

    {
  "exec_id": "e_xyz789...",
  "status": "running"
}

    exec_poll¶

    Poll for command execution output.

    Input: 

    {
  "exec_id": "e_xyz789...",
  "after_seq": 0
}

    Output: 

    {
  "messages": [
    {
      "seq": 1,
      "stream": "stdout",
      "data": "Hello, World!\n",
      "timestamp": "2024-01-15T10:30:01Z"
    }
  ],
  "complete": false,
  "exit_code": null
}

    exec_signal¶

    Send signal to running execution.

    Input: 

    {
  "exec_id": "e_xyz789...",
  "signal": "SIGTERM"
}

    Output: 

    {
  "status": "signaled"
}

    Filesystem Tools¶

    fs_read¶

    Read file contents from container.

    Input: 

    {
  "container_id": "c_abc123...",
  "path": "/workspace/file.txt"
}

    Output: 

    {
  "content": "file contents",
  "size": 13,
  "etag": "abc123def456",
  "mtime": "2024-01-15T10:30:00Z"
}

    fs_write¶

    Write file contents to container.

    Input: 

    {
  "container_id": "c_abc123...",
  "path": "/workspace/file.txt",
  "content": "new contents",
  "mode": 420,
  "create_parents": true,
  "if_match": "abc123def456"
}

    Output: 

    {
  "size": 12,
  "etag": "new123abc456"
}

    fs_delete¶

    Delete file or directory from container.

    Input: 

    {
  "container_id": "c_abc123...",
  "path": "/workspace/temp",
  "recursive": true
}

    Output: 

    {
  "status": "deleted",
  "path": "/workspace/temp"
}

    fs_list¶

    List directory contents in container.

    Input: 

    {
  "container_id": "c_abc123...",
  "path": "/workspace",
  "recursive": false
}

    Output: 

    {
  "entries": [
    {
      "path": "/workspace/file.txt",
      "type": "file",
      "size": 1234,
      "mtime": "2024-01-15T10:30:00Z",
      "mode": 420
    }
  ]
}

    Error Responses¶

    All tools may return errors in this format:

    {
  "error": {
    "code": -32000,
    "message": "Error description",
    "data": {
      "details": "Additional error information"
    }
  }
}

    See Error Handling for complete error code reference.
